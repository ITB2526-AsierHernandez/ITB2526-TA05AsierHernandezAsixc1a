Web Server Implementation with DMZ, Security Rules, and Web Development

Project Title: Secure Network Architecture Implementation (DMZ Deployment)

Executive Summary

This project involved the design and implementation of a segmented network architecture using a Demilitarized Zone (DMZ). The primary goal was to host a public-facing web server while ensuring the complete isolation and security of the internal local area network (LAN). By utilizing a dedicated firewall, I established strict traffic control policies to mitigate external threats and prevent lateral movement.

Technical Architecture & Segmentation

The network was structured into three distinct security zones based on a color-coded trust model:

  • GREEN Zone (LAN): The high-trust internal network where local clients reside.
  • ORANGE Zone (DMZ): The semi-trusted zone hosting the Web Server, isolated from the internal network.
  • RED Zone (WAN): The untrusted external network (Internet).

Key Technical Implementations

  • Firewall & Traffic Management:
    • Configured a perimeter firewall to manage inter-zone communication.
    • Implemented specific Access Control Lists (ACLs):
      • Allowed: RED → ORANGE (HTTP/HTTPS access).
      • Allowed: ORANGE → RED (System updates).
      • Blocked: RED → GREEN and ORANGE → GREEN (Preventing lateral movement).
  • Web Server Hardening & Security:
    • Encryption: Deployed a Web Server with HTTPS by generating and configuring custom SSL/TLS certificates.
    • Intrusion Prevention: Configured mod-evasive to protect against DDoS and brute-force attacks.
  • System Optimization:
    • Automated network interface management using nmcli scripts to disable redundant interfaces (enp0s8) at boot, reducing the attack surface.

Tech Stack

  • Security: Firewall Administration, SSL/TLS Certificates, mod-evasive (IPS).
  • Networking: DMZ Architecture, VLAN/Interface segmentation, TCP/IP.
  • Servers: Linux-based Web Servers (Apache), Client Workstations.
  • Tools: nmcli, Virtualization, Firewall Web GUI.

Core Competencies

  • Network Defense: Designing architectures that isolate critical assets.
  • Risk Mitigation: Implementing "Least Privilege" access at the network level.
  • Automation: Scripting system startups to maintain security posture.
